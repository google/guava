034/**
035 * Provides a fluent API for calculating <a
036 * href="http://en.wikipedia.org/wiki/Quantile">quantiles</a>.
037 *
038 * <h3>Examples</h3>
039 *
040 * <p>To compute the median:
041 *
042 * <pre>{@code
043 * double myMedian = median().compute(myDataset);
044 * }</pre>
045 *
046 * where {@link #median()} has been statically imported.
047 *
048 * <p>To compute the 99th percentile:
049 *
050 * <pre>{@code
051 * double myPercentile99 = percentiles().index(99).compute(myDataset);
052 * }</pre>
053 *
054 * where {@link #percentiles()} has been statically imported.
055 *
056 * <p>To compute median and the 90th and 99th percentiles:
057 *
058 * <pre>{@code
059 * Map<Integer, Double> myPercentiles =
060 *     percentiles().indexes(50, 90, 99).compute(myDataset);
061 * }</pre>
062 *
063 * where {@link #percentiles()} has been statically imported: {@code myPercentiles} maps the keys
064 * 50, 90, and 99, to their corresponding quantile values.
065 *
066 * <p>To compute quartiles, use {@link #quartiles()} instead of {@link #percentiles()}. To compute
067 * arbitrary q-quantiles, use {@link #scale scale(q)}.
068 *
069 * <p>These examples all take a copy of your dataset. If you have a double array, you are okay with
070 * it being arbitrarily reordered, and you want to avoid that copy, you can use {@code
071 * computeInPlace} instead of {@code compute}.
072 *
073 * <h3>Definition and notes on interpolation</h3>
074 *
075 * <p>The definition of the kth q-quantile of N values is as follows: define x = k * (N - 1) / q; if
076 * x is an integer, the result is the value which would appear at index x in the sorted dataset
077 * (unless there are {@link Double#NaN NaN} values, see below); otherwise, the result is the average
078 * of the values which would appear at the indexes floor(x) and ceil(x) weighted by (1-frac(x)) and
079 * frac(x) respectively. This is the same definition as used by Excel and by S, it is the Type 7
080 * definition in <a
081 * href="http://stat.ethz.ch/R-manual/R-devel/library/stats/html/quantile.html">R</a>, and it is
082 * described by <a
083 * href="http://en.wikipedia.org/wiki/Quantile#Estimating_the_quantiles_of_a_population">
084 * wikipedia</a> as providing "Linear interpolation of the modes for the order statistics for the
085 * uniform distribution on [0,1]."
086 *
087 * <h3>Handling of non-finite values</h3>
088 *
089 * <p>If any values in the input are {@link Double#NaN NaN} then all values returned are {@link
090 * Double#NaN NaN}. (This is the one occasion when the behaviour is not the same as you'd get from
091 * sorting with {@link java.util.Arrays#sort(double[]) Arrays.sort(double[])} or {@link
092 * java.util.Collections#sort(java.util.List) Collections.sort(List&lt;Double&gt;)} and selecting
093 * the required value(s). Those methods would sort {@link Double#NaN NaN} as if it is greater than
094 * any other value and place them at the end of the dataset, even after {@link
095 * Double#POSITIVE_INFINITY POSITIVE_INFINITY}.)
096 *
097 * <p>Otherwise, {@link Double#NEGATIVE_INFINITY NEGATIVE_INFINITY} and {@link
098 * Double#POSITIVE_INFINITY POSITIVE_INFINITY} sort to the beginning and the end of the dataset, as
099 * you would expect.
100 *
101 * <p>If required to do a weighted average between an infinity and a finite value, or between an
102 * infinite value and itself, the infinite value is returned. If required to do a weighted average
103 * between {@link Double#NEGATIVE_INFINITY NEGATIVE_INFINITY} and {@link Double#POSITIVE_INFINITY
104 * POSITIVE_INFINITY}, {@link Double#NaN NaN} is returned (note that this will only happen if the
105 * dataset contains no finite values).
106 *
107 * <h3>Performance</h3>
108 *
109 * <p>The average time complexity of the computation is O(N) in the size of the dataset. There is a
110 * worst case time complexity of O(N^2). You are extremely unlikely to hit this quadratic case on
111 * randomly ordered data (the probability decreases faster than exponentially in N), but if you are
112 * passing in unsanitized user data then a malicious user could force it. A light shuffle of the
113 * data using an unpredictable seed should normally be enough to thwart this attack.
114 *
115 * <p>The time taken to compute multiple quantiles on the same dataset using {@link Scale#indexes
116 * indexes} is generally less than the total time taken to compute each of them separately, and
117 * sometimes much less. For example, on a large enough dataset, computing the 90th and 99th
118 * percentiles together takes about 55% as long as computing them separately.
119 *
120 * <p>When calling {@link ScaleAndIndex#compute} (in {@linkplain ScaleAndIndexes#compute either
121 * form}), the memory requirement is 8*N bytes for the copy of the dataset plus an overhead which is
122 * independent of N (but depends on the quantiles being computed). When calling {@link
123 * ScaleAndIndex#computeInPlace computeInPlace} (in {@linkplain ScaleAndIndexes#computeInPlace
124 * either form}), only the overhead is required. The number of object allocations is independent of
125 * N in both cases.
126 *
127 * @author Pete Gillin
128 * @since 20.0
129 */

275    /**
276     * Computes the quantile value of the given dataset, performing the computation in-place.
277     *
278     * @param dataset the dataset to do the calculation on, which must be non-empty, and which will
279     *     be arbitrarily reordered by this method call
280     * @return the quantile value
281     */
282    public double computeInPlace(double... dataset) {
283      checkArgument(dataset.length > 0, "Cannot calculate quantiles of an empty dataset");
284      if (containsNaN(dataset)) {
285        return NaN;
286      }
287
288      // Calculate the quotient and remainder in the integer division x = k * (N-1) / q, i.e.
289      // index * (dataset.length - 1) / scale. If there is no remainder, we can just find the value
290      // whose index in the sorted dataset equals the quotient; if there is a remainder, we
291      // interpolate between that and the next value.
292
293      // Since index and (dataset.length - 1) are non-negative ints, their product can be expressed
294      // as a long, without risk of overflow:
295      long numerator = (long) index * (dataset.length - 1);
296      // Since scale is a positive int, index is in [0, scale], and (dataset.length - 1) is a
297      // non-negative int, we can do long-arithmetic on index * (dataset.length - 1) / scale to get
298      // a rounded ratio and a remainder which can be expressed as ints, without risk of overflow:
299      int quotient = (int) LongMath.divide(numerator, scale, RoundingMode.DOWN);
300      int remainder = (int) (numerator - (long) quotient * scale);
301      selectInPlace(quotient, dataset, 0, dataset.length - 1);
302      if (remainder == 0) {
303        return dataset[quotient];
304      } else {
305        selectInPlace(quotient + 1, dataset, quotient + 1, dataset.length - 1);
306        return interpolate(dataset[quotient], dataset[quotient + 1], remainder, scale);
307      }
308    }
309  }

389    /**
390     * Computes the quantile values of the given dataset, performing the computation in-place.
391     *
392     * @param dataset the dataset to do the calculation on, which must be non-empty, and which will
393     *     be arbitrarily reordered by this method call
394     * @return an unmodifiable, ordered map of results: the keys will be the specified quantile
395     *     indexes, and the values the corresponding quantile values. When iterating, entries in the
396     *     map are ordered by quantile index in the same order that the indexes were passed to the
397     *     {@code indexes} method.
398     */
399    public Map<Integer, Double> computeInPlace(double... dataset) {
400      checkArgument(dataset.length > 0, "Cannot calculate quantiles of an empty dataset");
401      if (containsNaN(dataset)) {
402        Map<Integer, Double> nanMap = new LinkedHashMap<>();
403        for (int index : indexes) {
404          nanMap.put(index, NaN);
405        }
406        return unmodifiableMap(nanMap);
407      }
408
409      // Calculate the quotients and remainders in the integer division x = k * (N - 1) / q, i.e.
410      // index * (dataset.length - 1) / scale for each index in indexes. For each, if there is no
411      // remainder, we can just select the value whose index in the sorted dataset equals the
412      // quotient; if there is a remainder, we interpolate between that and the next value.
413
414      int[] quotients = new int[indexes.length];
415      int[] remainders = new int[indexes.length];
416      // The indexes to select. In the worst case, we'll need one each side of each quantile.
417      int[] requiredSelections = new int[indexes.length * 2];
418      int requiredSelectionsCount = 0;
419      for (int i = 0; i < indexes.length; i++) {
420        // Since index and (dataset.length - 1) are non-negative ints, their product can be
421        // expressed as a long, without risk of overflow:
422        long numerator = (long) indexes[i] * (dataset.length - 1);
423        // Since scale is a positive int, index is in [0, scale], and (dataset.length - 1) is a
424        // non-negative int, we can do long-arithmetic on index * (dataset.length - 1) / scale to
425        // get a rounded ratio and a remainder which can be expressed as ints, without risk of
426        // overflow:
427        int quotient = (int) LongMath.divide(numerator, scale, RoundingMode.DOWN);
428        int remainder = (int) (numerator - (long) quotient * scale);
429        quotients[i] = quotient;
430        remainders[i] = remainder;
431        requiredSelections[requiredSelectionsCount] = quotient;
432        requiredSelectionsCount++;
433        if (remainder != 0) {
434          requiredSelections[requiredSelectionsCount] = quotient + 1;
435          requiredSelectionsCount++;
436        }
437      }
438      sort(requiredSelections, 0, requiredSelectionsCount);
439      selectAllInPlace(
440          requiredSelections, 0, requiredSelectionsCount - 1, dataset, 0, dataset.length - 1);
441      Map<Integer, Double> ret = new LinkedHashMap<>();
442      for (int i = 0; i < indexes.length; i++) {
443        int quotient = quotients[i];
444        int remainder = remainders[i];
445        if (remainder == 0) {
446          ret.put(indexes[i], dataset[quotient]);
447        } else {
448          ret.put(
449              indexes[i], interpolate(dataset[quotient], dataset[quotient + 1], remainder, scale));
450        }
451      }
452      return unmodifiableMap(ret);
453    }
454  }

456  /** Returns whether any of the values in {@code dataset} are {@code NaN}. */
457  private static boolean containsNaN(double... dataset) {
458    for (double value : dataset) {
459      if (Double.isNaN(value)) {
460        return true;
461      }
462    }
463    return false;
464  }

466  /**
467   * Returns a value a fraction {@code (remainder / scale)} of the way between {@code lower} and
468   * {@code upper}. Assumes that {@code lower <= upper}. Correctly handles infinities (but not
469   * {@code NaN}).
470   */
471  private static double interpolate(double lower, double upper, double remainder, double scale) {
472    if (lower == NEGATIVE_INFINITY) {
473      if (upper == POSITIVE_INFINITY) {
474        // Return NaN when lower == NEGATIVE_INFINITY and upper == POSITIVE_INFINITY:
475        return NaN;
476      }
477      // Return NEGATIVE_INFINITY when NEGATIVE_INFINITY == lower <= upper < POSITIVE_INFINITY:
478      return NEGATIVE_INFINITY;
479    }
480    if (upper == POSITIVE_INFINITY) {
481      // Return POSITIVE_INFINITY when NEGATIVE_INFINITY < lower <= upper == POSITIVE_INFINITY:
482      return POSITIVE_INFINITY;
483    }
484    return lower + (upper - lower) * remainder / scale;
485  }

512  /**
513   * Performs an in-place selection to find the element which would appear at a given index in a
514   * dataset if it were sorted. The following preconditions should hold:
515   *
516   * <ul>
517   *   <li>{@code required}, {@code from}, and {@code to} should all be indexes into {@code array};
518   *   <li>{@code required} should be in the range [{@code from}, {@code to}];
519   *   <li>all the values with indexes in the range [0, {@code from}) should be less than or equal
520   *       to all the values with indexes in the range [{@code from}, {@code to}];
521   *   <li>all the values with indexes in the range ({@code to}, {@code array.length - 1}] should be
522   *       greater than or equal to all the values with indexes in the range [{@code from}, {@code
523   *       to}].
524   * </ul>
525   *
526   * This method will reorder the values with indexes in the range [{@code from}, {@code to}] such
527   * that all the values with indexes in the range [{@code from}, {@code required}) are less than or
528   * equal to the value with index {@code required}, and all the values with indexes in the range
529   * ({@code required}, {@code to}] are greater than or equal to that value. Therefore, the value at
530   * {@code required} is the value which would appear at that index in the sorted dataset.
531   */
532  private static void selectInPlace(int required, double[] array, int from, int to) {
533    // If we are looking for the least element in the range, we can just do a linear search for it.
534    // (We will hit this whenever we are doing quantile interpolation: our first selection finds
535    // the lower value, our second one finds the upper value by looking for the next least element.)
536    if (required == from) {
537      int min = from;
538      for (int index = from + 1; index <= to; index++) {
539        if (array[min] > array[index]) {
540          min = index;
541        }
542      }
543      if (min != from) {
544        swap(array, min, from);
545      }
546      return;
547    }
548
549    // Let's play quickselect! We'll repeatedly partition the range [from, to] containing the
550    // required element, as long as it has more than one element.
551    while (to > from) {
552      int partitionPoint = partition(array, from, to);
553      if (partitionPoint >= required) {
554        to = partitionPoint - 1;
555      }
556      if (partitionPoint <= required) {
557        from = partitionPoint + 1;
558      }
559    }
560  }

562  /**
563   * Performs a partition operation on the slice of {@code array} with elements in the range [{@code
564   * from}, {@code to}]. Uses the median of {@code from}, {@code to}, and the midpoint between them
565   * as a pivot. Returns the index which the slice is partitioned around, i.e. if it returns {@code
566   * ret} then we know that the values with indexes in [{@code from}, {@code ret}) are less than or
567   * equal to the value at {@code ret} and the values with indexes in ({@code ret}, {@code to}] are
568   * greater than or equal to that.
569   */
570  private static int partition(double[] array, int from, int to) {
571    // Select a pivot, and move it to the start of the slice i.e. to index from.
572    movePivotToStartOfSlice(array, from, to);
573    double pivot = array[from];
574
575    // Move all elements with indexes in (from, to] which are greater than the pivot to the end of
576    // the array. Keep track of where those elements begin.
577    int partitionPoint = to;
578    for (int i = to; i > from; i--) {
579      if (array[i] > pivot) {
580        swap(array, partitionPoint, i);
581        partitionPoint--;
582      }
583    }
584
585    // We now know that all elements with indexes in (from, partitionPoint] are less than or equal
586    // to the pivot at from, and all elements with indexes in (partitionPoint, to] are greater than
587    // it. We swap the pivot into partitionPoint and we know the array is partitioned around that.
588    swap(array, from, partitionPoint);
589    return partitionPoint;
590  }

592  /**
593   * Selects the pivot to use, namely the median of the values at {@code from}, {@code to}, and
594   * halfway between the two (rounded down), from {@code array}, and ensure (by swapping elements if
595   * necessary) that that pivot value appears at the start of the slice i.e. at {@code from}.
596   * Expects that {@code from} is strictly less than {@code to}.
597   */
598  private static void movePivotToStartOfSlice(double[] array, int from, int to) {
599    int mid = (from + to) >>> 1;
600    // We want to make a swap such that either array[to] <= array[from] <= array[mid], or
601    // array[mid] <= array[from] <= array[to]. We know that from < to, so we know mid < to
602    // (although it's possible that mid == from, if to == from + 1). Note that the postcondition
603    // would be impossible to fulfil if mid == to unless we also have array[from] == array[to].
604    boolean toLessThanMid = (array[to] < array[mid]);
605    boolean midLessThanFrom = (array[mid] < array[from]);
606    boolean toLessThanFrom = (array[to] < array[from]);
607    if (toLessThanMid == midLessThanFrom) {
608      // Either array[to] < array[mid] < array[from] or array[from] <= array[mid] <= array[to].
609      swap(array, mid, from);
610    } else if (toLessThanMid != toLessThanFrom) {
611      // Either array[from] <= array[to] < array[mid] or array[mid] <= array[to] < array[from].
612      swap(array, from, to);
613    }
614    // The postcondition now holds. So the median, our chosen pivot, is at from.
615  }

617  /**
618   * Performs an in-place selection, like {@link #selectInPlace}, to select all the indexes {@code
619   * allRequired[i]} for {@code i} in the range [{@code requiredFrom}, {@code requiredTo}]. These
620   * indexes must be sorted in the array and must all be in the range [{@code from}, {@code to}].
621   */
622  private static void selectAllInPlace(
623      int[] allRequired, int requiredFrom, int requiredTo, double[] array, int from, int to) {
624    // Choose the first selection to do...
625    int requiredChosen = chooseNextSelection(allRequired, requiredFrom, requiredTo, from, to);
626    int required = allRequired[requiredChosen];
627
628    // ...do the first selection...
629    selectInPlace(required, array, from, to);
630
631    // ...then recursively perform the selections in the range below...
632    int requiredBelow = requiredChosen - 1;
633    while (requiredBelow >= requiredFrom && allRequired[requiredBelow] == required) {
634      requiredBelow--; // skip duplicates of required in the range below
635    }
636    if (requiredBelow >= requiredFrom) {
637      selectAllInPlace(allRequired, requiredFrom, requiredBelow, array, from, required - 1);
638    }
639
640    // ...and then recursively perform the selections in the range above.
641    int requiredAbove = requiredChosen + 1;
642    while (requiredAbove <= requiredTo && allRequired[requiredAbove] == required) {
643      requiredAbove++; // skip duplicates of required in the range above
644    }
645    if (requiredAbove <= requiredTo) {
646      selectAllInPlace(allRequired, requiredAbove, requiredTo, array, required + 1, to);
647    }
648  }
649
650  /**
651   * Chooses the next selection to do from the required selections. It is required that the array
652   * {@code allRequired} is sorted and that {@code allRequired[i]} are in the range [{@code from},
653   * {@code to}] for all {@code i} in the range [{@code requiredFrom}, {@code requiredTo}]. The
654   * value returned by this method is the {@code i} in that range such that {@code allRequired[i]}
655   * is as close as possible to the center of the range [{@code from}, {@code to}]. Choosing the
656   * value closest to the center of the range first is the most efficient strategy because it
657   * minimizes the size of the subranges from which the remaining selections must be done.
658   */
659  private static int chooseNextSelection(
660      int[] allRequired, int requiredFrom, int requiredTo, int from, int to) {
661    if (requiredFrom == requiredTo) {
662      return requiredFrom; // only one thing to choose, so choose it
663    }
664
665    // Find the center and round down. The true center is either centerFloor or halfway between
666    // centerFloor and centerFloor + 1.
667    int centerFloor = (from + to) >>> 1;
668
669    // Do a binary search until we're down to the range of two which encloses centerFloor (unless
670    // all values are lower or higher than centerFloor, in which case we find the two highest or
671    // lowest respectively). If centerFloor is in allRequired, we will definitely find it. If not,
672    // but centerFloor + 1 is, we'll definitely find that. The closest value to the true (unrounded)
673    // center will be at either low or high.
674    int low = requiredFrom;
675    int high = requiredTo;
676    while (high > low + 1) {
677      int mid = (low + high) >>> 1;
678      if (allRequired[mid] > centerFloor) {
679        high = mid;
680      } else if (allRequired[mid] < centerFloor) {
681        low = mid;
682      } else {
683        return mid; // allRequired[mid] = centerFloor, so we can't get closer than that
684      }
685    }
686
687    // Now pick the closest of the two candidates. Note that there is no rounding here.
688    if (from + to - allRequired[low] - allRequired[high] > 0) {
689      return high;
690    } else {
691      return low;
692    }
693  }
694
695  /** Swaps the values at {@code i} and {@code j} in {@code array}. */
696  private static void swap(double[] array, int i, int j) {
697    double temp = array[i];
698    array[i] = array[j];
699    array[j] = temp;
700  }
