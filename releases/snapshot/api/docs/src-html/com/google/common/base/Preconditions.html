022/**
023 * Static convenience methods that help a method or constructor check whether it was invoked
024 * correctly (whether its <i>preconditions</i> have been met). These methods generally accept a
025 * {@code boolean} expression which is expected to be {@code true} (or in the case of {@code
026 * checkNotNull}, an object reference which is expected to be non-null). When {@code false} (or
027 * {@code null}) is passed instead, the {@code Preconditions} method throws an unchecked exception,
028 * which helps the calling method communicate to <i>its</i> caller that <i>that</i> caller has made
029 * a mistake. Example: <pre>   {@code
030 *
031 *   /**
032 *    * Returns the positive square root of the given value.
033 *    *
034 *    * @throws IllegalArgumentException if the value is negative
035 *    *}{@code /
036 *   public static double sqrt(double value) {
037 *     Preconditions.checkArgument(value >= 0.0, "negative value: %s", value);
038 *     // calculate the square root
039 *   }
040 *
041 *   void exampleBadCaller() {
042 *     double d = sqrt(-1.0);
043 *   }}</pre>
044 *
045 * In this example, {@code checkArgument} throws an {@code IllegalArgumentException} to indicate
046 * that {@code exampleBadCaller} made an error in <i>its</i> call to {@code sqrt}.
047 *
048 * <h3>Warning about performance</h3>
049 *
050 * <p>The goal of this class is to improve readability of code, but in some circumstances this may
051 * come at a significant performance cost. Remember that parameter values for message construction
052 * must all be computed eagerly, and autoboxing and varargs array creation may happen as well, even
053 * when the precondition check then succeeds (as it should almost always do in production). In some
054 * circumstances these wasted CPU cycles and allocations can add up to a real problem.
055 * Performance-sensitive precondition checks can always be converted to the customary form:
056 * <pre>   {@code
057 *
058 *   if (value < 0.0) {
059 *     throw new IllegalArgumentException("negative value: " + value);
060 *   }}</pre>
061 *
062 * <h3>Other types of preconditions</h3>
063 *
064 * <p>Not every type of precondition failure is supported by these methods. Continue to throw
065 * standard JDK exceptions such as {@link java.util.NoSuchElementException} or
066 * {@link UnsupportedOperationException} in the situations they are intended for.
067 *
068 * <h3>Non-preconditions</h3>
069 *
070 * <p>It is of course possible to use the methods of this class to check for invalid conditions
071 * which are <i>not the caller's fault</i>. Doing so is <b>not recommended</b> because it is
072 * misleading to future readers of the code and of stack traces. See
073 * <a href="https://github.com/google/guava/wiki/ConditionalFailuresExplained">Conditional failures
074 * explained</a> in the Guava User Guide for more advice.
075 *
076 * <h3>{@code java.util.Objects.requireNonNull()}</h3>
077 *
078 * <p>Projects which use {@code com.google.common} should generally avoid the use of
079 * {@link java.util.Objects#requireNonNull(Object)}. Instead, use whichever of
080 * {@link #checkNotNull(Object)} or {@link Verify#verifyNotNull(Object)} is appropriate to the
081 * situation. (The same goes for the message-accepting overloads.)
082 *
083 * <h3>Only {@code %s} is supported</h3>
084 *
085 * <p>In {@code Preconditions} error message template strings, only the {@code "%s"} specifier is
086 * supported, not the full range of {@link java.util.Formatter} specifiers.
087 *
088 * <h3>More information</h3>
089 *
090 * <p>See the Guava User Guide on
091 * <a href="https://github.com/google/guava/wiki/PreconditionsExplained">using {@code
092 * Preconditions}</a>.
093 *
094 * @author Kevin Bourrillion
095 * @since 2.0
096 */

127  /**
128   * Ensures the truth of an expression involving one or more parameters to the calling method.
129   *
130   * @param expression a boolean expression
131   * @param errorMessageTemplate a template for the exception message should the check fail. The
132   *     message is formed by replacing each {@code %s} placeholder in the template with an
133   *     argument. These are matched by position - the first {@code %s} gets {@code
134   *     errorMessageArgs[0]}, etc. Unmatched arguments will be appended to the formatted message in
135   *     square braces. Unmatched placeholders will be left as-is.
136   * @param errorMessageArgs the arguments to be substituted into the message template. Arguments
137   *     are converted to strings using {@link String#valueOf(Object)}.
138   * @throws IllegalArgumentException if {@code expression} is false
139   * @throws NullPointerException if the check fails and either {@code errorMessageTemplate} or
140   *     {@code errorMessageArgs} is null (don't let this happen)
141   */
142  public static void checkArgument(
143      boolean expression,
144      @Nullable String errorMessageTemplate,
145      @Nullable Object... errorMessageArgs) {
146    if (!expression) {
147      throw new IllegalArgumentException(format(errorMessageTemplate, errorMessageArgs));
148    }
149  }

1124   *
1125   * if (guardExpression) {
1126   *    throw new BadException(messageExpression);
1127   * }
1128   *
1129   * refactored so that messageExpression is moved to a separate String-returning method.

1135   * The alternative natural refactorings into void or Exception-returning methods are much slower.

1142   * But the methods in this class want to throw different exceptions, depending on the args, so it

1268  /**
1269   * Substitutes each {@code %s} in {@code template} with an argument. These are matched by
1270   * position: the first {@code %s} gets {@code args[0]}, etc. If there are more arguments than
1271   * placeholders, the unmatched arguments will be appended to the end of the formatted message in
1272   * square braces.
1273   *
1274   * @param template a non-null string containing 0 or more {@code %s} placeholders.
1275   * @param args the arguments to be substituted into the message template. Arguments are converted
1276   *     to strings using {@link String#valueOf(Object)}. Arguments can be null.
1277   */
1278  // Note that this is somewhat-improperly used from Verify.java as well.
1279  static String format(String template, @Nullable Object... args) {
1280    template = String.valueOf(template); // null -> "null"
1281
1282    // start substituting the arguments into the '%s' placeholders
1283    StringBuilder builder = new StringBuilder(template.length() + 16 * args.length);
1284    int templateStart = 0;
1285    int i = 0;
1286    while (i < args.length) {
1287      int placeholderStart = template.indexOf("%s", templateStart);
1288      if (placeholderStart == -1) {
1289        break;
1290      }
1291      builder.append(template, templateStart, placeholderStart);
1292      builder.append(args[i++]);
1293      templateStart = placeholderStart + 2;
1294    }
1295    builder.append(template, templateStart, template.length());
1296
1297    // if we run out of placeholders, append the extra args in square braces
1298    if (i < args.length) {
